Create memorable, production-grade frontend interfaces that avoid generic AI aesthetics.

Before coding:
- Identify the purpose, audience, and technical constraints.
- Choose a bold aesthetic direction and keep it consistent.
- Decide what the memorable visual idea is before implementation starts.

Design goals:
- Build real working code, not mockups.
- Use Material UI components and theming first when working in this repository.
- Prefer distinctive typography, intentional spacing, strong composition, and clear visual hierarchy.
- Use motion sparingly but meaningfully.
- Build atmosphere with gradients, texture, shadow, layering, and depth rather than flat defaults.

Avoid:
- Generic AI-looking layouts.
- Overused purple-on-white styling.
- Default-feeling typography such as Arial, Roboto, Inter, or plain system stacks unless the existing product already depends on them.
- Cookie-cutter section patterns without a point of view.

Implementation guidance:
- Match the complexity of the code to the visual ambition.
- For maximal designs, support the look with rich structure and layered styling.
- For minimal designs, be precise with spacing, scale, and restraint.
- Preserve accessibility and responsiveness.
- When editing an existing interface, respect the product’s established structure while still improving craft and polish.
